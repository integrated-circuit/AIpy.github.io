Welcome to AIpy's project Wiki website!


This website is specially designed to explain and guide you through the process of installation and use of AIpy


I) What is AIpy, and what difference it from other robot projects?

AIpy is an open-source framework that joins the power of a Raspberry Pi and python used to control a DIY robotic arm It is meant to be fully editable and upgradeable, giving him new abilities to interact and help you. By his nature of open-source project, the source code and all the assets can be modified and decompiled to enhance this project. In contrary to other projects who only let assets open-source and keep technical information private, AIpy is under GNU General Public License v3.0, who lets YOU do almost anything YOU want with the project, except make closed source versions of AIpy.
AIpy is make by the people and for the people.


II) Duh, a simple robot built on Raspberry Pi!

In fact, it isn't a simple robot like that. AIpy could be better qualified as a Cobot. It's a complex term, but to simplify a Cobot is a robot designed to work and assist users in their tasks. With the help of sensors and cameras can the robot adjust his position to interact with the user and his environment. While these robots are often expensive because there are for professional-level tasks, the aim of AIpy is to be the alternative to give makers a cheap and open-source helper for their projects.
In conclusion, with this project, I aim to offer a cheap helping tool to the maker community that is easily scriptable and editable.


III) I like this project, how can I help it?

First, thank you very much! It takes time to develop and debug code! From your side, you can clone the project, build your own, perhaps find some bugs in the script or 3D models, and report them to us. You could also advertise the project on social media, there are many communities of makers on Internet, like r/raspberrypi or others.
There is also a Gitkraken Board linked to the repository, who can be found in Progress and ideas. Submit your suggestions in the issue tab in GitHub and see the progress as we (maybe) try to implement them.

IV) (Insert citation) doesn't work!

Calm down, Kiddo. Reviewing code isn't fast.
At first, make sure that you set up everything like mentioned. If setup is correct, then explain the bug in the Issue tab, with explanations how you encounter it (photos, text files, logs), what modification to the code did you (perhaps) do and how to reproduce it. After the incident is submitted, we will try to patch this error as quickly as possible.
